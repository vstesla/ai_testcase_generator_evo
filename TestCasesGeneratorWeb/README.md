# 智能测试用例与附件生成系统 (前端)

这是一个基于 React 和 Ant Design 构建的前端 Web 应用，主要用于对接后端 AI Agent 服务，实现测试集数据泛化、对抗攻击生成，以及存款协议/缴款通知书等指令附件的自动化排版与生成比对功能。

## ✨ 核心功能

系统包含两大核心功能模块，可通过顶部“功能类型”下拉框进行切换：

### 1. 测试集泛化
*   **功能描述**: 支持用户上传原始的 Excel 测试集配置文件，后端将自动调用内部 AI Agent 对测试数据进行泛化和对抗攻击生成，并返回处理后的 Excel 文件供下载。
*   **参数控制**:
    *   **泛化开关**: 决定是否对数据进行 AI 泛化扩充。
    *   **对抗生成开关**: 决定是否注入对抗性异常数据。

### 2. 指令附件生成 & 比对
*   **功能描述**: 针对具体的业务场景（如缴款通知书、各类他行定存协议、标的合同等），自动化生成排版精美的 PDF/Word 附件，并支持将生成的附件自动推送到后台解析引擎进行结构化内容比对，最后回显比对评测结果。
*   **支持的附件类型**:
    *   缴款通知书（需上传基础测试集 Excel 与 Word 模板）
    *   标的合同（需上传基础测试集 Excel 与 Word 模板）
    *   他行定存-线下必备条款
    *   他行定存-线上-不开具实证书必备条款
    *   他行定存-线上-开具实证书必备条款
    *   他行活期必备条款
*   **参数控制**:
    *   **文件数量**: 指定需要生成的文件份数。
    *   **泛化开关 & 对抗生成开关**: （对于需要测试集的场景）在生成附件前先对基础数据进行 AI 泛化/对抗处理。
    *   **评测比对开关**: 开启后，系统将在生成附件后自动轮询后端的 `/comparison_result` 接口，并在页面下方展示详尽的字段级解析评测报告（包含正确数、错误数、准确率等）。

### 3. 数据检索与历史回溯
*   **多维度查询工具栏**: 在“生成结果”和“比对结果”区域提供强大的联合查询能力，支持按“案例集ID”、“生成/比对状态”和“时间范围”进行精确筛选。
*   **比对详情弹窗**: 提供宽度为 1400px 的弹窗，方便用户通过唯一的 `TestCaseID` 查看结构化的比对明细，并在弹窗内直接导出对应的 Excel 分析报表。

## 🛠️ 技术栈

*   **框架**: React
*   **UI 组件库**: Ant Design (antd)
*   **样式处理**: CSS 
*   **数据导出**: `xlsx` (支持将生成结果和比对结果列表一键导出为 Excel)
*   **构建工具**: react-scripts

## 🚀 快速开始

### 1. 环境准备
请确保你的本地环境中已安装了 [Node.js](https://nodejs.org/) (建议版本 v14 及以上)。

### 2. 安装依赖
在项目根目录下打开终端，运行以下命令安装项目所需的全部依赖：

```bash
npm install
```

### 3. 启动开发服务器
依赖安装完成后，运行以下命令启动项目：

```bash
npm start
```
此时，浏览器会自动打开 `http://localhost:3000` 并展示系统页面。

### 4. 接口配置 (联调说明)
本项目采用了统一的 API 服务架构管理后端接口请求。默认请求的后端 API 基础路径配置在 `src/configs/apiConfig.js` 中：
```javascript
const ENV_CONFIG = {
  development: {
    API_BASE_URL: '/api',  // 开发环境使用相对路径，通过代理转发
    AI_QA_BASE_URL: '/api' // 开发环境使用相对路径，通过代理转发
  },
  // ...
};
```
在进行前后端联调时，请确保你的前端通过配置了正确的代理（如 `package.json` 中的 `proxy`）指向后端服务（如 `http://127.0.0.1:8000`）。所有的接口端点已在 `apiConfig.js` 中的 `API_ENDPOINTS` 统一定义，业务层的接口调用（如 `AiTestCaseGeneratorService.js`）全部基于 `unifiedApiService` 发起，内置了 Token 注入、缓存控制、错误拦截等能力。

## 💡 特色交互设计

1.  **智能联动表单**: 根据选择的功能类型，动态展示/隐藏附件类型下拉框、文件上传框，并自动锁定不相关的开关状态（如测试集泛化模式下自动禁用评测开关）。
2.  **丝滑动画体验**: 比对结果列表区域采用了基于 CSS 的平滑展开/收起过渡动画，提升用户体验。
3.  **多文件自动合并下载**: 接口返回多个附件下载链接时，前端已做兼容处理，点击一次下载按钮即可循环触发所有文件的下载。对于单文件场景，修复了强制 ZIP 打包的问题，支持直出 PDF 原文件。
4.  **实时状态反馈与轮询**: 对于需要耗时的 AI 评测任务和大批量文件生成（如 300 份指令），后台基于 Celery 队列进行异步处理。前端实现了基于统一 `TestCaseID` 轮询的异步状态拉取机制，让用户能够实时感知后台队列的“处理中”状态，并在完成后立刻呈现结果。
5.  **高并发状态管理**: 引入基于纯函数状态更新的机制彻底解决 React 闭包陷阱问题；重构了历史脏数据的过滤策略，限制了单次全量刷新触发的定时器风暴，有效避免了高并发场景下的 504 Timeout 请求雪崩。
